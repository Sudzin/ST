# Руководство пользователя системы передачи файлов

## Введение
Данная система предназначена для безопасной передачи файлов между клиентом и сервером по протоколу TCP (WebSocket). Система обеспечивает аутентификацию пользователей, загрузку и скачивание файлов, а также администрирование процесса передачи.

## Требования к системе
- Современный веб-браузер (Chrome, Firefox, Edge, Safari).
- Подключение к сети.

## Роли пользователей
1. **Пользователь (User)**: Может загружать файлы на сервер и скачивать доступные файлы.
2. **Администратор (Admin)**: Имеет доступ к панели управления, может просматривать логи, управлять пользователями и историей передач.

---

## Инструкция для Пользователя

### 1. Регистрация и Вход
1. Откройте приложение в браузере.
2. Если у вас нет аккаунта, нажмите ссылку "Register" (Регистрация).
3. Введите имя пользователя и пароль.
4. После регистрации войдите в систему, используя свои учетные данные.

### 2. Загрузка файла (Upload)
1. На главной панели (User Dashboard) нажмите на область загрузки или перетащите файл в пунктирную область.
2. Нажмите кнопку "Start Upload".
3. Следите за индикатором прогресса.
4. После успешной загрузки появится сообщение "File uploaded successfully!".

### 3. Скачивание файла (Download)
1. В разделе "Available Files" (Доступные файлы) отображается список всех загруженных на сервер файлов.
2. Нажмите на иконку скачивания (стрелка вниз) напротив нужного файла.
3. Файл будет сохранен на вашем устройстве.

### 4. Статус подключения
- Индикатор в правом верхнем углу показывает статус подключения к серверу:
  - **Connected (Зеленый)**: Связь с сервером установлена.
  - **Disconnected (Красный)**: Связь отсутствует. Попробуйте обновить страницу.

## Инструкция для Администратора

### 1. Вход в панель администратора
1. Войдите в систему под учетной записью с правами администратора (role: admin).
2. Вы будете перенаправлены на Admin Dashboard.

### 2. Просмотр статистики (Overview)
- На вкладке "Overview" отображается:
  - Общее количество переданных файлов.
  - Общий объем переданных данных.
  - Количество активных подключений (максимум 3).
  - Текущая скорость передачи данных.

### 3. Управление журналом передач (Transfers)
1. Перейдите на вкладку "Transfers".
2. Здесь отображается список всех передач файлов с указанием имени файла, размера, времени и статуса.
3. **Просмотр деталей**: Нажмите на строку передачи, чтобы увидеть детальный график скорости и пакеты данных.
4. **Редактирование**: Нажмите иконку карандаша, чтобы изменить имя файла или статус передачи.
5. **Удаление**: Нажмите иконку корзины, чтобы удалить запись о передаче и сам файл с сервера. Это действие необратимо.

### 4. Просмотр системных логов (Logs)
1. Перейдите на вкладку "Logs".
2. Здесь фиксируются все события системы: входы пользователей, начало и конец загрузки, ошибки и отключения.
3. **Фильтрация**: Используйте поля "Filter by User" и "Filter by Action" для поиска конкретных событий.
4. **Управление**: Вы можете редактировать или удалять записи логов при необходимости.

### 5. Ограничения системы
- Максимальное количество одновременных подключений: 3.
- При превышении лимита новые пользователи получат сообщение об ошибке.

---

## Устранение неполадок

**Ошибка "Server full"**
- Сервер достиг лимита в 3 подключения. Подождите, пока кто-то из пользователей отключится, или закройте лишние вкладки.

**Ошибка подключения**
- Проверьте интернет-соединение.
- Обновите страницу.
- Если проблема сохраняется, обратитесь к администратору сервера.

**Файл не загружается**
- Убедитесь, что файл не поврежден.
- Проверьте статус подключения (должен быть "Connected").
